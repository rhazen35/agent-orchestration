# Records

This directory tracks the *why* behind technical and specification decisions. In a spec-driven development workflow, understanding the reasoning behind changes is as important as the changes themselves. Without this context, future contributors (or agents) risk repeating past mistakes or undoing deliberate choices.

## Contents

- `ADR-XXX.md` — Architecture Decision Records. Each file documents a significant technical choice: the context, the decision made, alternatives considered, and consequences. Use `templates/records/ADR-001-template.md` as the template.
- `spec-evolution.log` — A chronological audit trail of requirement and specification changes, including the reason for each change and what was impacted.
